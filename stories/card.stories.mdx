# Cards
Cards are components that you can use within your project to display content or highlight specific information related to a single topic.

Cards by nature should be easy to read and relevant to the information they're displaying. You cann add specific details within cards to make this information stand out. For example adding
an: `image`, `title`, `description` or `cardTags`.

Let's have a look at the various card setups you can have within Poggle.

# Basic Card
This is perhaps the most basic card you can have within Poggle. This card by default has a basic content padding with a `cardShadow` that renders the `description` you place inside the component.

<Canvas>
    <Story name="Basic Card" args={{ description: "This is your card text"}}>
        {Template.bind({})}
    </Story>
</Canvas>

Poggle automatically limits your `description` length to 200 characters, you can change this by changing the `desLength` prop within the component. You can also change the font family by editing the
`cardFont` prop for all items. You may also edit the `description` font size by using the `fnSize` prop, the `desSize` is rendered using rem not px, so just enter a value, for example 2.

Additionally you can edit your cards `width` and `height`. By default the `width` is set to 25rem and the height is auto.

# Card With An Image
Within Poggle you can easily customise a card component to include an `image`. You can add information to the card using the `description` attribute as shown above. But you may also want an eye catching
`title`. You can use the `titleSize` prop to change the text size of your card `title`.

You can also limit the amount of characters a title can render using the `titleLength` property. Additionally, you may decide you require rounded corners for this `Card`. You can achieve this by using the `cardRadius` property, this is also measured in rem units.

# Card With Tags
Within Poggle you can dynamically add tags to your `Card` component. You can add tags in an array by using the `cardTags` property. By default, you're limited to 4 tags per card.
You can also edit the tags position on the page by using the `tagsPos` for either `top`, `middle` or `bottom`, this is `middle` by default.

With the addition of tags you may need to add padding to your `description`, you can use the `desMy` to add margin vertically or `desMx` to add margin horizontally (measured in rem units).

By default your tags have a gray background and a black text colour. You can easily edit these properties by using the following: `tagsBgColor` for the background colour (default is lightgray)
`tagsTxtColor` for the colour of the text (default is black).

#### You can also edit the tags padding, fontsize and font-weight. [Look at the "Create A Card" section for more](/docs/components-cards--create-a-card).

# Responsive Cards
With our card's system you can easily set up different card widths at different breakpoints. Each breakpoint has a default if you choose to leave it blank. The breakpoints are in ascending order
`xsWidth`, `smWidth`, `mdWidth` and `lgWidth`. Give it a try, resize your browser and see this card change to fit your screen.

By default all `cards` have build in responsive defaults. To override these defaults edit the aforementioned properties above.

<Canvas>
  <Story name="Responsive Cards" args={{
    image: "https://images.pexels.com/photos/1154198/pexels-photo-1154198.jpeg?auto=compress&cs=tinysrgb&dpr=2&h=750&w=1260",
    title: "A Responsive Card!",
    titleAlign: "center",
    width: 30,
    xsWidth: 20,
    smWidth: 28,
    mdWidth: 33,
  }}>{Template.bind({})}</Story>
</Canvas>
